# graduation-project — コーデ共有アプリ

## サービス概要
- ファッション好きが自分のコーディネートを気軽に投稿・共有できるアプリ。
- シンプルな画像投稿・閲覧に特化し、ユーザー同士がスタイルを参考にできる。
- 将来的にはコーデ投稿に対してブランド側が「似合うアイテム提案」を返せる機能も拡張予定。

## このサービスへの思い・作りたい理由
私はアパレル業界で働き、お客様に似合うアイテムを考えることが好きでした。
今でもファッションを見ること、コーデを考えることは生活の一部です。
また自分自身がブランド立ち上げをしているため、将来的にはユーザーが投稿したコーデに対して“ショップ・ブランド側が提案できる接点”を作りたいという思いもあります。
まずは核心となる「コーデ共有部分」をMVPとして形にしたいと考えています。

## ユーザー層について
- **一般ユーザー（10〜30代のファッション好き）**
  - SNSでコーデを見る習慣があり、投稿する文化が根付いている
  - WEARは機能が多く、気軽さに欠けると感じる層がいる
  - 「シンプルで使いやすいコーデ投稿アプリ」へのニーズがある

- **将来的なターゲット：小規模ブランド・個人ブランド運営者**
  - （提案機能を追加した際のターゲット）
  - ユーザーのスタイルを見た上で商品提案ができると購入率が上がる
  - コミュニティ作り・顧客理解のツールとして利用価値が高い

## サービスの利用イメージ
1. ユーザーがコーデ写真を投稿する（画像・タグ・テキスト）
2. 他のユーザーはそのコーデを一覧や詳細で閲覧できる
3. 気に入った投稿を参考にして、自分のスタイリングのヒントを得られる

**ユーザーが得られる価値**
- シンプルで投稿しやすい
- “自分のコーデ日記”として使える
- 他人のリアルなコーデから学べる

**将来的な拡張**
- ブランド側の「あなたに似合うアイテム提案」が受けられる体験へ拡張予定
ユーザーの投稿データ（ブランド・色・テイストなど）から好みをAIが分析し、ブランド側がそのユーザーに合う新作アイテムやおすすめを直接提案できる仕組みです。ユーザーは受け取るだけで良く、ブランド側は効率的に　ターゲット層へアプローチできるため、お互いに価値のある機能になると考えています。
※AI分析機能はMVPでは実装せず、将来的な拡張として想定しています。まずはユーザーが投稿したコーデ情報（色・ブランド・テイストなど）をデータとして蓄積し、後からAIやブランド側の提案ロジックに活用できるようにする計画です。

## ユーザーの獲得について
- 一般ユーザー：Instagram / TikTokで投稿 → アプリへ誘導
- ブランド運営者：個人ブランドやショップ店員インフルエンサーに先行利用を依頼

## サービスの差別化ポイント・推しポイント
- **WEARとの差別化**：WEARは投稿のハードルが高い。投稿に必要な工程が多く、写真のクオリティが求められやすい点、そしてファッションに強いユーザー層が多く新規ユーザーが投稿しづらい雰囲気があるためです。私のアプリでは、顔出し不要・最低限の入力で投稿できる・日常の簡単な記録として使える、という気軽さを重視しています。
- **独自性**：アパレル経験者の視点から「コーデの投稿・閲覧」に必要な機能だけを抽出。将来的にブランド→ユーザーの提案機能を追加できる土台を用意。

## 機能候補
### MVP（最初に作るもの）
- ユーザー登録 / ログイン（Devise）
- コーデ投稿（画像1枚、テキスト、タグ）
- 投稿一覧 / 詳細表示
- 投稿編集 / 削除（CRUD）

### 本リリースで追加したい機能
- いいね機能、フォロー機能、タグ検索、コメント、ユーザーページ（投稿一覧）
- 将来的：ブランド側アカウント、提案コメント機能、提案まとめページ

## 使用する技術スタック
- フレームワーク：Ruby on Rails 7
- 言語：Ruby 3.1
- DB：PostgreSQL
- デプロイ：Render または Fly.io
- 使用予定ライブラリ：devise、ActiveStorage、rubocop、tailwindcss or bootstrap

## 実装イメージ
画像投稿は ActiveStorage を利用。投稿の CRUD が中心のため初学者でも構築しやすい。MVPでは複雑な非同期通信や推論ロジックは使わず、後から段階的に追加する方針。
